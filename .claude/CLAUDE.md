## What This Is

Terraform provider for Gravitee APIM (API Management). Most code is **generated by Speakeasy** from an OpenAPI spec ‚Äî do not manually edit files marked with `// Code generated by Speakeasy`. Custom code is preserved within marked regions (e.g., `// BEGIN GRAVITEE CLOUD INIT`).

## Build & Test Commands

```bash
# Build
go build

# Code generation (requires speakeasy CLI)
make speakeasy

# Lint (OAS + Terraform fmt)
make lint
make lint-fix          # fix Terraform formatting

# Sync OAS from upstream APIM
make sync-oas

# Unit tests (no APIM required)
make unit-tests

# Acceptance tests (requires running APIM)
make acceptance-tests

# Example tests (requires running APIM + local provider build)
make examples-tests

# Run a single acceptance test
APIM_USERNAME=admin APIM_PASSWORD=admin APIM_SERVER_URL=http://localhost:30083/automation \
  TF_ACC=1 go test -count=1 -v -run TestAccApiv4Resource ./tests/acceptance

# Run a single unit test
go test -count=1 -v -run TestSomething ./internal/...

# Local Kind cluster for APIM
make start-cluster                          # full mode
APIM_MINIMAL=true make start-cluster        # minimal (faster, management API only)
make stop-cluster
```

## Architecture

### Code Generation Flow
1. OpenAPI spec lives in `schemas/automation-api-oas.yaml` (with overlays in `schemas/overlays/`)
2. Speakeasy generates SDK (`internal/sdk/`) and provider code (`internal/provider/`)
3. Config: `.speakeasy/gen.yaml` and `.speakeasy/gen.lock`

### Provider Structure (`internal/provider/`)
- **Resources** (4): `apim_apiv4`, `apim_application`, `apim_shared_policy_group`, `apim_subscription`
- **Data sources** (4): mirror resources for read-only access
- Each resource has: `*_resource.go` (CRUD), `*_resource_sdk.go` (type mapping)
- `provider.go` ‚Äî provider config, auth (bearer/cloud/basic), schema
- `utils.go` ‚Äî HTTP logging, state merging, plan refresh helpers
- `reflect/` and `typeconvert/` ‚Äî Terraform ‚Üî SDK type conversion
- `planmodifiers/` ‚Äî custom plan modification logic per type
- `validators/` ‚Äî custom validation logic
- `customtypes/` ‚Äî RFC3339 datetime, trimmed string types

### SDK (`internal/sdk/`)
- Generated Go client: `apis.go`, `applications.go`, `subscriptions.go`, `sharedpolicygroups.go`
- Models in `models/operations/` and `models/components/`

### Tests (`tests/`)
- `tests/acceptance/` ‚Äî resource lifecycle tests (create, read, update, delete, import)
- `tests/examples/` ‚Äî runs actual Terraform configs from `examples/`
- All acceptance/example tests require a running APIM instance

### Authentication
Provider supports three auth methods (checked in order):
1. Service account token (`APIM_SA_TOKEN`)
2. Gravitee Cloud token (`APIM_CLOUD_TOKEN`)
3. Basic auth (`APIM_USERNAME` / `APIM_PASSWORD`)

## Changing Provider Schema Behavior

**To change provider schema behavior** (plan modifiers, computed fields, validators, defaults, suppress diffs, etc.), add or update entries in `schemas/overlays/` and regenerate with `make speakeasy`. Never edit generated files directly.

Key Speakeasy overlay annotations:
- `x-speakeasy-param-computed: true` ‚Äî mark a field as computed
- `x-speakeasy-param-suppress-computed-diff: true` ‚Äî add `SuppressDiff` plan modifier (only works for Unknown plan values)
- `x-speakeasy-plan-modifiers: MyModifier` ‚Äî reference a custom plan modifier function from `internal/planmodifiers/<type>planmodifier/`

Custom plan modifiers, validators, and types in `internal/planmodifiers/`, `internal/validators/`, and `internal/customtypes/` are **non-generated** and can be edited freely.

## Key Conventions

- Generated code is the majority of the codebase ‚Äî custom changes go in marked regions or non-generated files
- Import IDs are JSON-encoded composite keys (see test files for format)
- APIM compatibility: 4.9+
- CI runs acceptance tests across a matrix of Terraform versions (1.9, 1.14) √ó APIM versions (4.9, 4.10, 4.11)
- Issues tracked at github.com/gravitee-io/issues with tag `project: GKO`
